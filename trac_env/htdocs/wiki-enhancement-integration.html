üöÄ Integration Instructions

Step 1: CSS File Placement

The enhancement file wiki-professional-enhancement.css should be placed in your Trac environment's htdocs directory:

trac_env/htdocs/wiki-professional-enhancement.css

Step 2: Update Trac Configuration

Add the CSS file to your trac_env/conf/trac.ini configuration:

[inherit]
htdocs_dir = /path/to/trac_env/htdocs

[project]
name = Your Project Name

# Add to existing CSS includes or create new section
[theme]
htdocs_location = 
css = professional-theme.css, wiki-professional-enhancement.css

Step 3: Template Customization (Optional)

For the search placeholder text, you can customize the search input in your templates:

<input type="text" name="q" placeholder="Search Wiki‚Ä¶" />

Step 4: Browser Cache

Clear your browser cache and restart Trac to see the changes:

# Restart Trac server
sudo systemctl restart apache2  # or your web server

üîß Customization Options

Adjust Content Width

To change the max-width of Wiki content:

#content.wiki {
  max-width: 1200px !important; /* Default: 1000px */
}

Modify Color Scheme

The enhancement uses CSS variables from the professional theme. You can override specific colors:

:root {
  --accent-primary: #6366f1; /* Change from blue to indigo */
  --text-primary: #0f172a;   /* Darker text */
}

Typography Adjustments

Fine-tune typography sizes:

.wikipage h1 {
  font-size: 32px !important; /* Default: 28px */
}

.wikipage p {
  font-size: 16px !important; /* Default: 15px */
}

üêõ Troubleshooting

Styles Not Applying

  • Check that the CSS file path is correct in trac.ini
  • Verify file permissions allow web server access
  • Clear browser cache and hard refresh (Ctrl+F5)
  • Check browser developer tools for CSS loading errors

Layout Issues

  • Ensure the professional theme base is loaded first
  • Check for CSS conflicts with custom themes
  • Verify viewport meta tag for responsive behavior

Font Issues

  • Confirm internet access for Google Fonts loading
  • Consider self-hosting Inter font for offline environments
  • Check browser console for font loading errors
